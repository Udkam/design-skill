# Brief: Dashboard

Use `$craft-frontend-design` to redesign a port operations dashboard that dispatchers use during repeated shifts to compare berth status, vessel risk, and maintenance blockers.

## Expected Routing

- `interface-taxonomy.md`: dashboard / operational console.
- `platform-adapters.md`: dashboard constraints.
- `visual-systems.md`: dense layout and status colors.
- `quality-gates.md`: scan speed and accessibility.

## Expected Output

- Starts with operational status, not a hero.
- Uses compact tables/lists and semantic status.
- Highlights abnormal states.
- Covers loading, empty, error, disabled/current states.
- Avoids marketing-style layout.
