# Brief: Game UI

Use `$craft-frontend-design` to design a small browser game UI for a timing-based tile puzzle. The UI should support play state, score, pause/restart, and immediate feedback without covering the board.

## Expected Routing

- `interface-taxonomy.md`: game / interactive toy UI.
- `platform-adapters.md`: game UI constraints.
- `motion-and-interaction.md`: immediate feedback and reduced motion.
- `quality-gates.md`: focus, state, and responsive checks.

## Expected Output

- Board is primary.
- Score/progress and controls are readable.
- Covers loading, error, disabled, selected, pending, and success states.
- Motion is purposeful and reducible.
- No marketing or dashboard patterns.
