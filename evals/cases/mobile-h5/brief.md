# Brief: Mobile H5

Use `$craft-frontend-design` to implement a mobile-first H5 flow for reserving a maintenance pickup window. It should be short, touch-friendly, and resilient to unavailable times.

## Expected Routing

- `interface-taxonomy.md`: mobile H5.
- `platform-adapters.md`: mobile constraints.
- `content-and-microcopy.md`: concise task copy.
- `quality-gates.md`: touch target and state coverage.

## Expected Output

- Starts with available pickup windows.
- Uses thumb-friendly controls.
- Covers unavailable, loading, empty, error, disabled, and success states.
- Avoids desktop sidebar and landing-page hero patterns.
