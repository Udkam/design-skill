# Brief: Personal Site Interactive

Use `$craft-frontend-design` to refine a personal website that mixes portfolio, project routes, a small game entry, repo links, a console surface, and a lightweight pet/toy interaction.

## Expected Routing

- `interface-taxonomy.md`: portfolio / personal site plus game / interactive toy UI.
- `platform-adapters.md`: web, responsive, floating controls, and mobile touch targets.
- `motion-and-interaction.md`: purposeful motion with JS/canvas/video reduced-motion handling.
- `quality-gates.md`: shared island blast radius, state coverage, accessibility, and public hygiene.

## Expected Output

- Preserve the personal identity system instead of converting the site into a generic SaaS hero.
- Keep Projects, Game, Repo, and Console entries visible and understandable.
- Keep the pet/toy interaction bounded, optional, and responsive.
- Stop JS animation loops when reduced motion is active.
- Keep mobile tap targets at least 44px.
