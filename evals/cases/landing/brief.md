# Brief: Landing Page

Use `$craft-frontend-design` to redesign a landing page for a repair-logistics product used by regional appliance service teams. The page should not look like a generic AI SaaS template.

## Expected Routing

- `interface-taxonomy.md`: marketing / landing.
- `visual-systems.md`: product artifact, typography, color semantics.
- `content-and-microcopy.md`: replace vague AI copy.
- `motion-and-interaction.md`: reduced motion for transitions.
- `quality-gates.md`: first viewport and anti-template check.

## Expected Output

- First viewport states product category, user, proof, and primary CTA.
- Uses product-specific objects such as repair routes, parts holds, and technician windows.
- Includes realistic loading/error/disabled state copy.
- Uses a product artifact image with alt text.
- Avoids purple/blue gradient hero, blobs, generic three-card feature grid, and vague AI copy.
