# Changelog

## 0.3.0 - Plugin skeleton and negative evals

- Improved audit precision with app/public-release/docs profiles, contextual placeholder handling, safer `Coming soon` classification, docs-aware secret examples, JS reduced-motion checks, and shared component blast-radius hints.
- Added a synthetic personal-site interactive eval and a negative personal-site-as-SaaS-hero eval based on abstract real-project feedback.
- Added reproducible negative eval fixtures for template-heavy heroes, accessibility gaps, motion without reduced motion, mini-program platform mismatch, and dashboard-as-marketing failures.
- Added `scripts/build_plugin_package.py` to generate the local/plugin-ready skeleton from the canonical skill.
- Added `scripts/package_release.py` to create local skill and plugin release archives.
- Added `plugins/craft-frontend-design/` with `.codex-plugin/plugin.json`, synchronized skill files, and local package README.
- Added narrow audit checks for dashboard-as-marketing and mini-program/mobile platform mismatch cases.
